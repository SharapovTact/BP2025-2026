2. Merge
Т.к считывание построчное, у меня оно выполняется в теле цикла, после сравнения считывается
элемент из того файла, в котором он оказался меньше. Но чтобы войти в этот цикл мне нужно
уже что-то сравнивать, поэтому я защищённо считал ровно 1 символ из каждого файла. 

EOF в основном цикле нужен для того, чтобы обработать случай когда у меня сливаются два файла
с единственным символом, т.к я уже к этому моменту их считал - каретка будет смотреть на конец
строки. Значит нам подходит EOF.

Ниже EOF можно увидеть в блоках кода, которые копируют хвост (который не с чем сравнивать). Чтобы
не проверять на наличие этого хвоста - я копирую файлы по EOF, значит если у меня не будет хвоста,
то все файлы не пройдут эту проверку.
